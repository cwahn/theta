## Release v0.1.0-alpha.53

- **Fix: WASM `ask()` no longer reimplements the reply protocol** — The macro-generated `ask()` on WASM actor ref classes previously duplicated the reply-dispatch logic from `ActorRef::ask()` but was missing the remote fallback path (`oneshot::Receiver<(Peer, Vec<u8>)>` downcast + `PEER.sync_scope` deserialization). Remote asks silently returned the wrong type or panicked. Fixed by delegating all ask arms to `self.inner.ask(inner).await` — a single canonical path that handles both local and remote actors.

- **Fix: Unit struct message types emit `null` not `{}`** — `#[derive(TsType)]` on unit structs (e.g. `Ping`) was generating `export interface X {}`, causing TypeScript callers to pass `{}`. `serde_wasm_bindgen` expects `null` for unit structs. Fixed: unit structs now emit `export type X = null`.

- **Fix: WASM PEER scope cleared during JS→Rust deserialization** — When a WASM import task yields at an async recv point, the `PEER` thread-local stays set. A synchronous JS→Rust deserialization of a message carrying an `ActorRef` field then took the DTO branch instead of preserving the live JS class. Fixed by adding `theta::ts::from_js_value()` (mirroring the existing `to_js_value()`) that wraps deserialization in `PEER.clear_scope`.

- **New: `ref-roundtrip` WASM/TS integration test suite** — Six end-to-end tests (T1–T6) verifying native↔WASM `ActorRef` round-trips: plain ask, `ActorRef`-in-message-field, unit struct messages, `Vec<ActorRef>` returns, and ask-through-relayed-ref. Integrated into `workflow/test_examples.sh`.

## Release v0.1.0-alpha.52

- **Fix: PEER task-local leak on WASM** ‚Äî `ActorRef` fields inside `View` data and `ask()` return values were serialized as plain DTO objects instead of live WASM classes when accessed across async yield points. Root cause: WASM single-threaded runtime shares a `thread_local` cell; the remote monitor reader runs inside `PEER.scope(peer, async { loop { recv... } })`, leaving PEER set when it yields, causing other `spawn_local` tasks to inherit the wrong peer context during `serde_wasm_bindgen::to_value`. Fixed by adding `clear_scope` to `compat_task_local!` and wrapping all WASM‚ÜíJS serialization calls in `PEER.clear_scope`.

- **New: `theta-ts` React hooks** ‚Äî `useActorView` and `useActorRef` typed hooks built on `useSyncExternalStore`; `CachedStream` with ref-counted shared subscription; `createStream` factory.

- **New: `vite-plugin-theta` actor aliases** ‚Äî per-actor hook aliases (`useXxxView`, `useXxxRef`) generated at build time; umbrella crate edition capped to `2021`; infinite rebuild loop fixed by excluding generated umbrella `src/` from wasm-pack watch.

- **Web-chat example redesign** ‚Äî symmetric `ChatManager` with named bind (`bind("manager")`); guest discovery via `lookupChatManager`; View-as-truth reactive state (no imperative `ask(ListRooms)` on the hot path); auto-select first room; guests can also create rooms; full peer key display.

- **Fix: duplicate React instances** ‚Äî added `resolve.dedupe: ['react', 'react-dom']` to Vite config to prevent nested `node_modules` conflict from `file:` linked `theta-ts`.

- **theta-lint** ‚Äî workspace linter with 5 rules: `use-toplevel`, `sort-use`, `cfg-order`, `use-group-order`, `path-depth`; wired into pre-commit hook and CI.

- **Pre-commit hook** ‚Äî `.githooks/pre-commit` runs `cargo fmt --check`, `cargo clippy`, and `theta-lint` before every commit.
